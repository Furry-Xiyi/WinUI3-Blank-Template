# WinUI3 模板 - 本地化使用指南

本模板支持多语言本地化功能，目前已配置中文（简体、繁体）和英文。

## 📁 文件结构

```
Strings/
├── zh-CN/
│   └── Resources.resw    # 简体中文资源文件
├── zh-TW/
│   └── Resources.resw    # 繁体中文资源文件
├── en-US/
│   └── Resources.resw    # 英文资源文件
└── 本地化使用指南.md      # 本文档

Services/
└── LocalizationHelper.cs  # 本地化辅助类
```

## 🎯 使用方法

### 1. 在 XAML 中使用本地化

#### 方法 A：使用 `x:Uid` 属性（推荐）

这是 WinUI 3 的标准本地化方式，适用于大多数控件。

**简单文本控件：**

```xml
<!-- XAML 中 -->
<TextBlock x:Uid="SettingsPageTitle" Text="设置" />
```

```xml
<!-- Resources.resw 中 -->
<data name="SettingsPageTitle.Text" xml:space="preserve">
  <value>设置</value>
</data>
```

**SettingsCard 卡片本地化：**

```xml
<!-- XAML 中 -->
<controls:SettingsCard x:Uid="ThemeSettingsCard">
    <controls:SettingsCard.HeaderIcon>
        <FontIcon Glyph="&#xE706;"/>
    </controls:SettingsCard.HeaderIcon>
    <!-- 卡片内容 -->
</controls:SettingsCard>
```

```xml
<!-- Resources.resw 中 -->
<data name="ThemeSettingsCard.Header" xml:space="preserve">
  <value>应用主题</value>
</data>
<data name="ThemeSettingsCard.Description" xml:space="preserve">
  <value>选择浅色、深色或跟随系统</value>
</data>
```

**SettingsExpander 展开卡片本地化：**

```xml
<!-- XAML 中 -->
<controls:SettingsExpander x:Uid="MaterialSettingsExpander" IsExpanded="False">
    <controls:SettingsExpander.HeaderIcon>
        <FontIcon Glyph="&#xE2B1;"/>
    </controls:SettingsExpander.HeaderIcon>
    <controls:SettingsExpander.Items>
        <!-- 子项内容 -->
    </controls:SettingsExpander.Items>
</controls:SettingsExpander>
```

```xml
<!-- Resources.resw 中 -->
<data name="MaterialSettingsExpander.Header" xml:space="preserve">
  <value>背景材质</value>
</data>
<data name="MaterialSettingsExpander.Description" xml:space="preserve">
  <value>选择 Mica 或 Acrylic</value>
</data>
```

**多属性本地化示例：**

```xml
<!-- XAML 中 -->
<Button x:Uid="SaveButton" 
        Content="保存"
        ToolTipService.ToolTip="保存设置" />
```

```xml
<!-- Resources.resw 中 -->
<data name="SaveButton.Content" xml:space="preserve">
  <value>保存</value>
</data>
<data name="SaveButton.[using:Microsoft.UI.Xaml.Controls]ToolTipService.ToolTip" xml:space="preserve">
  <value>保存设置</value>
</data>
```

### 2. 在 C# 代码中使用本地化

使用 `LocalizationHelper` 类获取本地化字符串：

```csharp
using WinUI3.Services;

// 获取本地化字符串
string message = LocalizationHelper.GetString("ErrorMessage");

// 在对话框中使用
var dialog = new ContentDialog
{
    Title = LocalizationHelper.GetString("DialogTitle"),
    Content = LocalizationHelper.GetString("DialogContent"),
    PrimaryButtonText = LocalizationHelper.GetString("ConfirmButton"),
    CloseButtonText = LocalizationHelper.GetString("CancelButton")
};
```

## 📝 添加新的本地化字符串

### 步骤 1：在资源文件中添加键值对

在 `Strings/zh-CN/Resources.resw` 中添加：

```xml
<data name="MyNewString" xml:space="preserve">
  <value>我的新字符串</value>
</data>
```

在 `Strings/en-US/Resources.resw` 中添加对应的英文：

```xml
<data name="MyNewString" xml:space="preserve">
  <value>My New String</value>
</data>
```

### 步骤 2：在 XAML 或代码中使用

**XAML：**
```xml
<TextBlock x:Uid="MyNewString" />
```

**C#：**
```csharp
string text = LocalizationHelper.GetString("MyNewString");
```

## ⚠️ 常见问题与注意事项

### 1. ❌ 错误：使用不支持的语法

```xml
<!-- ❌ 错误：不要使用这种语法 -->
<controls:SettingsCard x:Uid="MyCard"
                      x:Uid:Description="MyDescription" />
```

```xml
<!-- ✅ 正确：只使用一个 x:Uid -->
<controls:SettingsCard x:Uid="MyCard" />
```

### 2. 资源文件命名规则

- 使用 `.` 分隔 Uid 和属性名：`UidName.PropertyName`
- 对于附加属性，使用完整命名空间：`UidName.[using:Namespace]PropertyName`
- 保持命名一致性，建议使用 `区域_元素_属性` 格式

### 3. 项目配置注意事项

**不需要手动添加 PRIResource 项！**

.NET SDK 会自动包含所有 `.resw` 文件，手动添加会导致 NETSDK1022 错误。

```xml
<!-- ❌ 不要添加这个 -->
<ItemGroup>
  <PRIResource Include="Strings\**\*.resw" />
</ItemGroup>
```

如果遇到重复项错误，请移除手动添加的 PRIResource 配置。

### 4. 动态创建的 UI 元素

对于在代码中动态创建的 UI 元素，必须使用 `LocalizationHelper`：

```csharp
var card = new SettingsCard
{
    Header = LocalizationHelper.GetString("DynamicCard.Header"),
    Description = LocalizationHelper.GetString("DynamicCard.Description")
};
```

### 5. 语言切换

如果实现了语言切换功能，切换后需要调用：

```csharp
LocalizationHelper.Reset();
```

然后重启应用以使更改生效。

## 📚 命名规范建议

### 页面元素命名

```
格式：PageName_ElementName.PropertyName

示例：
- SettingsPage_Title.Text
- HomePage_WelcomeText.Text
- SettingsPage_ThemeCard.Header
```

### 通用元素命名

```
格式：Category_ElementName

示例：
- Nav_Home.Content
- Dialog_Confirm.Content
- Button_Save.Content
```

### 卡片命名

```
格式：PageName_CardPurpose

示例：
- SettingsPage_ThemeCard
- SettingsPage_LanguageExpander
- HomePage_WelcomeCard
```

## 🌍 支持的语言

当前模板已配置以下语言：

- 简体中文 (zh-CN)

### 添加新语言

1. 在 `Strings` 文件夹下创建新的语言文件夹（如 `ja-JP`）
2. 复制 `en-US/Resources.resw` 到新文件夹
3. 翻译所有字符串值
4. 重新生成项目

## 💡 最佳实践

1. **保持同步**：添加新字符串时，同时更新所有语言的资源文件
2. **使用描述性键名**：键名应清晰表达用途，如 `SettingsPage_ThemeCard.Header`
3. **避免硬编码**：所有用户可见的文本都应使用资源文件
4. **测试多语言**：切换语言后测试所有页面，确保文本正确显示
5. **提供回退**：`LocalizationHelper` 已实现回退机制，找不到资源时返回键名
6. **注释重要资源**：在资源文件中为复杂或特殊的字符串添加注释
